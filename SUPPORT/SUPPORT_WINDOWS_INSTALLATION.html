Windows Installation (OptiPlex 3020)

Offline setup and project clone procedure.

Installation Procedure

  1. Connect the Optiplex PC to the Internet via Ethernet cable. It is needed to copy the project from GitHub (phase2). Check connection by running ping 8.8.8.8 in the powershell. If no connection established - follow the docs/sharing-Internet-from-devPC-to-Optiplex.md instruction.
  2. Copy "/windows-install" folder to the Optiplex Desktop.
  3. Open PowerShell as an admin. Right-click Start -> PowerShell (Admin).
  4. In PowerShell run cd C:\Users\Dell\Desktop\windows-install.
  5. In PowerShell run Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned; press Y; press Enter.
  6. In PowerShell run .\phase1-install-deps-offline.ps1.
  7. If there will be git-related error, close PowerShell and open a new one, then repeat steps 3, 4, 6.
  8. After successful dependencies installation restart the computer.
  9. Start the Docker app from the Desktop (if not started automatically), press skip everywhere, wait until Docker Engine is started.
  10. Open PowerShell as an admin.
  11. If Docker says so, open PowerShell as admin and run wsl --update, restart the Docker app and wait until you see that Docker Engine is started. If there is no requirement to update the WSL, skip this step.
  12. In PowerShell run cd C:\Users\Dell\Desktop\windows-install.
  13. In PowerShell run .\phase2-git-manage-clone.ps1. Press Enter twice.
  14. The project is copied at C:\Projects\HOTWHEELS.
  15. Go to C:\Projects\HOTWHEELS folder (if not yet there). To build the HotWheels project container stack run next command: 
    docker compose --env-file .env.docker build
  16. Spin the container stack by running the next command in the terminal: 
    17. docker compose --env-file .env.docker up
    Wait for all the images to be downloaded.
  17. Delete /mysql_data folder
  18. Reconfigure the Ethernet connection with the next settings (refer to PC Setup Guide in Software Architecture docs): 
    IP address: 192.168.1.2
Subnet mask: 255.255.255.0
Default gateway: 192.168.1.1
  19. Repeat step 16.
  20. Congrats, the HotWheels project is up and running!
  21. To run the project after reboot, open the terminal as admin and type next commands: 
    cd C:\Projects\HOTWHEELS
docker compose --env-file .env.docker up
  22. Access the server via browser at address 192.168.1.2:3101/HW

Enabling Kiosk on start

  1. Press Win + R, run shell:startup.
  2. Copy start-kiosk.bat from the windows-install folder into the Startup folder.

Troubleshooting

  • npm ci fails during build - ensure package-lock.json is present in the repository (remove it from .gitignore if necessary). Docker copies the lock file into the image so dependencies install deterministically.
  • Entrypoint reports no such file - Windows converts shell scripts to CRLF. Reconvert with tr -d '\r' < scripts/docker-entrypoint.sh > scripts/docker-entrypoint.tmp && mv scripts/docker-entrypoint.tmp scripts/docker-entrypoint.sh && chmod +x scripts/docker-entrypoint.sh, or disable Git CRLF conversion (git config core.autocrlf false) before rebuilding.
  • MySQL container stuck "unhealthy" - adjust the healthcheck to use mysqladmin ping -h localhost -uroot -p$$MYSQL_ROOT_PASSWORD. If HOST is not allowed appears, reset the DB volume (docker compose down -v) so the seed script re-creates grants, or grant 'hotwheels'@'%' manually.
  • Can't run the project after turning PC on, "Can't connect to mysql" - Start Docker Desktop manually until you see "Engine running"
  • daemon connection problem - in .env.docker change the environment variable HW_SERVER_IP_HOST=192.168.1.2
Follow each step in order for a clean install on the OptiPlex.
